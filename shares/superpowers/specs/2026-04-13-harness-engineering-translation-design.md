# 设计文档：Harness Engineering 翻译实战

## 一、 项目概览
本文旨在分享一种基于 Harness Engineering（编排工程）思想的翻译工作流。通过将大型书籍翻译任务分解为可控的原子化任务，并利用 Gemini CLI 和结构化的 System Prompt，实现高质量、一致性强的书籍翻译。

## 二、 核心理念
- **任务降维**：将“翻译一本书”的长链条任务，降维拆解为“翻译一个片段”的原子化任务。
- **一致性控制**：通过精心编排的 System Prompt（术语表、风格定义），在单任务驱动模式下保持长文本的语义连贯性。
- **可控执行**：利用 Gemini CLI 的标准化能力，确保每个片段的翻译结果具备高度的一致性和可复现性。

## 三、 技术实现方案
- **引擎**：Gemini CLI
- **输入**：Markdown/TXT 格式书籍内容
- **核心组件**：
    1. **System Prompt**：包含术语库、语气词规范、上下文风格限定。
    2. **分片策略**：简单的文本切片逻辑，保证每个片段在 LLM 上下文窗口的最佳处理长度。
    3. **CLI 工作流**：单命令调用的原子化执行模式。

## 四、 内容结构
1. **引言**：从“颗粒度焦虑”出发，定义 Harness Engineering 的本质。
2. **Prompt 编排**：展示如何通过结构化定义 System Prompt 来约束翻译风格。
3. **CLI 实战**：梳理“分片 -> 执行 -> 校对”的自动化微流程。
4. **思考**：总结一致性控制与工具链编排的工程艺术。

## 五、 质量保证标准
- CLI 指令需具备可复现性。
- Prompt 设计需具备模块化特征，便于维护。
- 内容需展示真实的交互过程（而非仅展示结果）。
